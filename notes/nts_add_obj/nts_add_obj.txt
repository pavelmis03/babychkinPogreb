0) при создании предмета, который должен быть твердым для игрока, буханки или NPC, если это 
	простой предмет, с которым нельзя взаимодействовать, мы добавляем описание этого предмета в
	CONST_MAP_SOLIDOBJ, там подробно описано, какие у него характеристики. После этого можно создать
	в нужном месте карты объект	obj_simple_wall и присвоить ему имя name = "name", чтобы
	этот объект получил свои характеристики из константного массива. Все)
1) при добавлении препятствия, через которое нельзя пройти, нужно добавить его в массив player_solidObj
	во всех персонажах
2) при добавлении объекта, с которым персонаж может взаимодействовать, необходимо создать ему переменную
	can_interact, которая будет сообщать, что с предметом можно взаимодействовать и будет проверяться персонажем 
	при нажатии игроком Е
3) если объект нужно сохранять, прописать ему need_sv = 1, если он не перемещается по комнатам и need_sv = 2 в ином случае
4) при добавлении новых игровых контроллеров, их нужно вносить в события UE12 и UE13 объекта obj_ctrl_gm
//5) при добавлении системных объектов/контроллеров, их нужно вносить в событие UE15 объекта obj_ctrl_gm
6) при добавлении постоянных контроллеров с need_sv = 2, им нужно создать событие UE0, если у них есть структуры данных
	(словари, списки и т.д.) их нужно создавать в create в виде обычной переменной, а в UE0 нормально и читать из строки
	которую передает контроллер сохранений при загрузке. Пример см. в obj_ctrl_gm_inv в UE0 и в create
7) при добавлении объектов, которые надо сохранять, создавать у них массив nonSvVar = [],
	в который заносятся переменные типа init, которые не нужно сохранять для конкретного объекта, 
	а также см. nts_savesInfo, там могут быть доп. правила
8) ВАЖНО: при создании объекта, который должен сохраняться и при этом может перемещаться по комнатам, 
	необходимо создать для него постоянный контроллер c need_sv = 2 (как, например, obj_ctrl_gm_playerStatus),
	и сохранять именно его
9) ПРИ ДОБАВЛЕНИИ ПЕРСОНАЖА: в create obj_ctrl_gm_playerStatus добавить название объекта в массив 
	player_arr
